<objective>
Add project root configuration to the RSpec formatter so users can control where test.json is written.

Purpose: Without configuration, the formatter writes relative to Dir.pwd, which breaks in monorepo or workspace setups where tests run from a subdirectory. Users need to set the project root via environment variable (CFG-01) or RSpec configuration (CFG-02), following the same patterns established by the PHPUnit and Pytest reporters.

Output: Modified formatter with project root resolution, RSpec configuration registration, and tests proving both configuration methods work.
</objective>

  <action>
Modify the formatter to resolve project root from three sources with this precedence: RSpec config > env var > Dir.pwd.

In `reporters/rspec/lib/tdd_guard_rspec.rb`, add an RSpec.configure block that registers the `tdd_guard_project_root` setting:

```ruby
RSpec.configure do |config|
  config.add_setting :tdd_guard_project_root, default: nil
end
```

This goes after the existing `require_relative` line.

In `reporters/rspec/lib/tdd_guard_rspec/formatter.rb`, modify the `write_results` method to use a `project_root` helper instead of hardcoded `Dir.pwd`:

1. Add a private `project_root` method that checks (in order):
   - `RSpec.configuration.tdd_guard_project_root` (if non-nil and non-empty string)
   - `ENV['TDD_GUARD_PROJECT_ROOT']` (if non-nil and non-empty string)
   - `Dir.pwd` (fallback, preserves existing behavior)

2. Update `write_results` to use `project_root` instead of `Dir.pwd`:

   ```ruby
   def write_results(output_data)
     data_dir = File.join(project_root, ".claude", "tdd-guard", "data")
     FileUtils.mkdir_p(data_dir)
     output_path = File.join(data_dir, "test.json")
     File.write(output_path, JSON.generate(output_data))
   end
   ```

3. The `project_root` method should be simple and direct -- no path validation beyond checking non-empty. Keep it minimal like the Pytest reporter (which just checks absolute path and cwd-within-root). For our case, skip validation entirely -- trust the user's input. The formatter should not crash if the path is invalid; FileUtils.mkdir_p will fail with a clear error.

Do NOT add path traversal validation, absolute path checks, or cwd-within-root checks. Keep it simple -- the RSpec formatter is invoked by the user in their own project, not in a sandbox. Follow the Pytest pattern of minimal validation.
</action>
